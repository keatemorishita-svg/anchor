# 评论区神器 v2 升级策略

> 从"伪装系统"升级为"识别与对齐系统"

---

## 一、当前系统状态 (v1)

```
        用户选择维度
        ↓
  [谁在说] → 4 种人设 (20+女/30+女/20+男/30+男)
  [说什么] → 4 种话题 (项目/观察/感悟/行业)
  [为什么发] → 4 种目的 (记录/引流/互动/刷存在)
  [什么时候发] → 4 种节奏 (早上/中午/傍晚/晚上)
  [语气] → 4 种 (谦逊/轻松/调皮/温柔)
  [心情] → 6 种
        ↓
  系统扮演选中的人设 → 生成文案/评论
```

**核心问题**：系统在"扮演别人"——20+女、30+男等不同人设。这是伪装，长期会穿帮。

---

## 二、目标架构 (v2)

```
        输入目标内容（对方发的帖子/视频文案）
        ↓
  ① 作者人设识别 ← 新增：自动判断对方是哪种人
        ↓
  ② 我的固定人设 ← 从 4 种改为 1 种：「AI 连续创业者」
        ↓
  ③ 表达适配层   ← 新增：根据对方类型选择说话方式
        ↓
  ④ 信号生成     ← 沿用现有评论生成逻辑
        ↓
        输出评论
```

**核心改变**：我不变，我识别你，我用你能听懂的方式跟你说话。

---

## 三、新旧对比

| | v1（当前） | v2（目标） |
|---|---|---|
| 我的人设 | 4 种可选，每次切换 | 1 种固定：AI 连续创业者 |
| 对方识别 | 不做 | 自动识别 5 种作者类型 |
| 表达方式 | 硬绑定到人设 | 根据对方类型动态适配 |
| 主帖模式 | 有（保留） | 保留，从 4 人设简化为 1 固定 |
| 评论区模式 | 有 | 核心升级对象 |
| 用户操作 | 手动选维度 | 粘贴内容，自动识别 + 匹配 |

## 四、5 种作者类型识别规则

### 识别逻辑（关键词匹配 → 自动判断）

| 作者类型 | 关键词 | 我的角色 | 评论策略 |
|---|---|---|---|
| content_creator | 结构/方法/认知/写作/表达 | 同行补充者 | 共鸣 → 细节引用 → 经验补充 → 轻延展 |
| tool_builder | workflow/工具/系统/自动化/功能 | 使用者+提问者 | 使用感 → 细节引用 → 坑位信号 → 技术提问 |
| ai_operator | 变现/副业/客户/赚钱/效率 | 实操验证者 | 结果判断 → 场景补充 → 现实问题 |
| trend_thinker | 趋势/未来/行业/变化/判断 | 一线观察者 | 判断 → 一线观察 → 微反差 → 轻提问 |
| explorer | 默认（无明确特征） | 轻指导+同路人 | 轻共鸣 → 正反馈 → 轻补充 |

### 代码实现（可直接加入 server.py）

```python
AUTHOR_PERSONAS = {
    "content_creator": {
        "keywords": ["结构", "方法", "认知", "写作", "表达", "框架", "内容"],
        "my_role": "同行补充者",
        "strategy": "共鸣+补充",
        "rule": """对方是内容创作者。你不是教他，而是"我也在做，有一些实践感受"。
输出结构：共鸣 → 细节引用 → 经验补充 → 轻延展（可选提问）""",
    },
    "tool_builder": {
        "keywords": ["workflow", "工具", "系统", "自动化", "功能", "流程", "实现"],
        "my_role": "使用者+提问者",
        "strategy": "使用+提问",
        "rule": """对方是工具开发者。你不是夸他，而是"我用过/踩过坑"。
输出结构：使用感 → 细节引用 → 坑位信号 → 技术提问""",
    },
    "ai_operator": {
        "keywords": ["变现", "副业", "客户", "赚钱", "效率", "收益"],
        "my_role": "实操验证者",
        "strategy": "验证+现实问题",
        "rule": """对方是AI变现玩家。你不是讨论理念，而是"这个能不能跑通"。
输出结构：结果判断 → 场景补充 → 现实问题（可选提问）""",
    },
    "trend_thinker": {
        "keywords": ["趋势", "未来", "行业", "变化", "判断"],
        "my_role": "一线观察者",
        "strategy": "判断+一线观察",
        "rule": """对方是趋势型博主。你不是跟风，而是"我在一线的观察"。
输出结构：判断 → 一线观察 → 微反差 → 轻提问""",
    },
    "explorer": {
        "keywords": [],
        "my_role": "轻指导+同路人",
        "strategy": "鼓励+轻补充",
        "rule": """对方是新手探索者。鼓励为主，不要给建议。
输出结构：轻共鸣 → 正反馈 → 轻补充""",
    },
}


def detect_author_persona(text):
    """从内容文本自动识别作者类型"""
    for pt, cfg in AUTHOR_PERSONAS.items():
        if pt == "explorer":
            continue
        for kw in cfg["keywords"]:
            if kw in text:
                return pt
    return "explorer"
```

## 五、固定人设定义

```python
MY_IDENTITY = """我是谁：AI 行业连续创业者，被同行称为"老师"。沉稳、有阅历，不追逐热点。
发评论的唯一理由：观察到一个有意思的现象，觉得值得说一句。
语气：有阅历的人在茶桌上随口提起一件事。说完就完，不总结，不说破。
禁区：不提自己的成就、头衔、经历。不讲道理，不给建议。不装专家。"""
```

这与当前 `male-30plus` 人设基本相同——可以直接复用。

## 六、改动范围

### 6.1 server.py 改动

| 改动 | 说明 | 复杂度 |
|---|---|---|
| 新增 `AUTHOR_PERSONAS` 字典 | 5 种作者类型 + 识别规则 | 低 |
| 新增 `detect_author_persona()` | 关键词匹配自动识别 | 低 |
| 新增 `MY_IDENTITY` 常量 | 固定人设 | 低 |
| 修改 `assemble_prompt()` | 评论区模式加入作者识别 + 固定人设 + 表达适配 | 中 |
| 新增 `/api/detect` 端点（可选） | 前端可以预览识别结果 | 低 |

### 6.2 index.html 前端改动

| 改动 | 说明 | 复杂度 |
|---|---|---|
| 评论区粘贴内容后自动调用识别 | 显示"识别结果：内容创作者"标签 | 低 |
| 用户可手动覆盖识别结果 | 如果识别错了可以改 | 低 |
| 移除评论区"谁在说"维度切换 | 人设固定，不需要选了 | 低 |

### 6.3 主帖模式 (post mode)

| 现状 | v2 |
|---|---|
| 4 种人设可选 | 简化为 1 种固定人设 |
| "谁在说"维度保留但只有 1 个选项 | 或直接隐藏该维度 |

## 七、分阶段实施计划

### Phase 1：评论区升级（核心，1-2 小时）

1. 在 `server.py` 加入 `AUTHOR_PERSONAS` + `detect_author_persona()`
2. 修改 `assemble_prompt()`，评论区模式自动识别作者类型
3. 修改评论生成 prompt 结构：固定人设 + 作者类型 → 适配表达
4. 前端增加识别结果展示（小标签）
5. 测试 5 种作者类型的生成效果

### Phase 2：主帖模式简化（30 分钟）

1. 将 4 种 persona 替换为固定人设
2. 隐藏或移除"谁在说"维度选择
3. 保留其他维度（话题/目的/节奏/语气/心情）

### Phase 3：反馈闭环（后续，可选）

1. 记录每条评论的回复率
2. 根据回复率反向调整策略权重
3. 实现 `策略 → 命中率 → 强化学习` 循环

## 八、风险评估

| 风险 | 影响 | 缓解 |
|---|---|---|
| 关键词匹配过于简单，识别不准 | 错配策略，评论不对味 | 保留手动覆盖入口；后续可升级为 LLM 识别 |
| 固定人设不一定最适合用户 | 评论风格单一 | 固定人设可配置（.env 里定义） |
| 主帖模式从 4 人设变为 1 个 | 失去多样性 | 主帖模式用户量小，影响有限 |

## 九、决策要点

**立即执行如果**：
- 评论区是主要使用场景
- 你认同"我不变，我识别你"的理念
- 愿意接受一个更简单但更真实的系统

**暂缓执行如果**：
- 4 种人设的多样性对你很重要（主帖场景）
- 想先观察更多真实使用数据再决定
